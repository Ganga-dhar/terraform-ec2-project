🚀 Terraform Dynamic Blocks, Conditional Expressions, and Splat Expressions Explained

These are powerful Terraform language features that help you write reusable, DRY, and flexible Infrastructure as Code.

1️⃣ Dynamic Blocks

A dynamic block allows you to generate repeated nested configuration blocks programmatically instead of manually writing them multiple times.

❌ Without Dynamic Block
resource "aws_security_group" "web" {
  name = "web-sg"

  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["10.0.0.0/16"]
  }

  ingress {
    from_port   = 443
    to_port     = 443
    protocol    = "tcp"
    cidr_blocks = ["10.0.0.0/16"]
  }

  ingress {
    from_port   = 8080
    to_port     = 8080
    protocol    = "tcp"
    cidr_blocks = ["10.0.0.0/16"]
  }
}
✅ With Dynamic Block
variable "ingress_ports" {
  default = [80, 443, 8080]
}

resource "aws_security_group" "web" {
  name = "web-sg"

  dynamic "ingress" {
    for_each = var.ingress_ports

    content {
      from_port   = ingress.value
      to_port     = ingress.value
      protocol    = "tcp"
      cidr_blocks = ["10.0.0.0/16"]
    }
  }
}
Another Example – Multiple EBS Volumes
variable "ebs_volumes" {
  default = [
    {
      device = "/dev/sdf"
      size   = 50
    },
    {
      device = "/dev/sdg"
      size   = 100
    }
  ]
}

resource "aws_instance" "server" {
  ami           = "ami-xxxx"
  instance_type = "t3.medium"

  dynamic "ebs_block_device" {
    for_each = var.ebs_volumes

    content {
      device_name = ebs_block_device.value.device
      volume_size = ebs_block_device.value.size
    }
  }
}
When to Use Dynamic Blocks

✅ Security Group Rules
✅ EBS Volumes
✅ Route53 Records
✅ ALB Listener Rules
✅ IAM Statements
✅ Kubernetes manifests with repeated nested blocks

2️⃣ Conditional Expressions

Terraform supports if/else logic using the ternary operator.

Syntax:

condition ? true_value : false_value
Example 1 – Environment Based Instance Size
variable "environment" {
  default = "prod"
}

resource "aws_instance" "app" {
  instance_type = var.environment == "prod" ? "t3.large" : "t3.micro"
}
Example 2 – Enable Deletion Protection Only in Production
resource "aws_db_instance" "mysql" {
  deletion_protection =
    var.environment == "prod" ? true : false
}

Can be simplified:

deletion_protection = var.environment == "prod"
Example 3 – Optional Resource Creation
resource "aws_instance" "bastion" {
  count = var.enable_bastion ? 1 : 0
}
Example 4 – Dynamic Tags
tags = {
  Backup = var.environment == "prod" ? "Enabled" : "Disabled"
}
Common Uses

✅ Resource creation (count)
✅ Environment-specific sizing
✅ Optional modules
✅ Dynamic naming and tagging
✅ Feature flags

3️⃣ Splat Expressions

A splat expression is used to extract attributes from multiple resources.

Example Setup
resource "aws_instance" "web" {
  count = 3

  ami           = "ami-xxxx"
  instance_type = "t3.micro"
}
Traditional Method
output "instance_ids" {
  value = [
    aws_instance.web[0].id,
    aws_instance.web[1].id,
    aws_instance.web[2].id
  ]
}
Using Splat Expression
output "instance_ids" {
  value = aws_instance.web[*].id
}

Output:

[
  "i-12345",
  "i-67890",
  "i-99999"
]
Get Public IPs
output "public_ips" {
  value = aws_instance.web[*].public_ip
}
Module Output Example
output "bucket_names" {
  value = module.s3[*].bucket_name
}
Working with Optional Resources
resource "aws_instance" "bastion" {
  count = var.enable_bastion ? 1 : 0
}

output "bastion_ip" {
  value = aws_instance.bastion[*].public_ip
}

If disabled:

[]

instead of causing an index error.

Combining All Three Features
resource "aws_security_group" "web" {
  name = "web-sg"

  dynamic "ingress" {
    for_each =
      var.environment == "prod"
      ? [80,443,8080]
      : [80]

    content {
      from_port = ingress.value
      to_port   = ingress.value
      protocol  = "tcp"
      cidr_blocks = ["0.0.0.0/0"]
    }
  }
}

output "sg_ids" {
  value = aws_security_group.web[*].id
}
Interview Questions
Q1: Why use Dynamic Blocks?

To avoid repetitive nested blocks and make configurations reusable.

Q2: Difference between for_each and Dynamic Block?
Feature	for_each	Dynamic Block
Creates resources	✅ Yes	❌ No
Creates nested blocks	❌ No	✅ Yes
Works inside resources	Limited	Yes
Q3: Difference between Splat and For Expressions?

Splat

aws_instance.web[*].id

For Expression

[
  for i in aws_instance.web : i.id
]

for expressions are more flexible because they support filtering and transformations.

Example:

[
  for i in aws_instance.web :
  i.public_ip
  if i.instance_state == "running"
]

🚀 Terraform Tip: Dynamic Blocks, Conditional Expressions & Splat Expressions

These three Terraform features can dramatically simplify your Infrastructure as Code.

✅ Dynamic Blocks → Generate repeated nested blocks like security group rules or EBS volumes.

✅ Conditional Expressions → Implement environment-specific logic:

instance_type =
var.env == "prod"
? "t3.large"
: "t3.micro"

✅ Splat Expressions → Easily retrieve attributes from multiple resources:

aws_instance.web[*].public_ip

Using these features effectively results in:

✔ Less repetitive code
✔ More reusable modules
✔ Easier environment customization
✔ Cleaner and more maintainable Terraform configurations

Small Terraform language features often make the biggest difference in large-scale infrastructure projects.
